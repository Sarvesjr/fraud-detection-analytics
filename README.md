# Fraud Detection Analytics (PaySim Dataset)

## 📌 Overview

This project analyzes financial transactions using the PaySim dataset (~6.3M records) to detect fraudulent behavior.

It demonstrates an end-to-end data workflow including:

* Data Quality Engineering
* Feature Engineering
* SQL Analytics
* Dashboard Visualization
* Data Pipeline Design

---

## 🧠 Key Highlights

* Built a rule-based fraud detection system
* Performed large-scale data validation (6M+ rows)
* Designed engineered features for fraud detection
* Created SQL-based business insights
* Developed interactive Power BI dashboard

---

## 🛠 Tech Stack

* Python (Pandas)
* SQL (MySQL)
* Power BI
* Excel

---

## 📊 Dataset

* Source: PaySim (Kaggle)
* Records: 6.36 Million
* Fraud Rate: ~0.13%

---

## 📂 Project Structure

* `fraud_detection1.ipynb` → Data cleaning & feature engineering
* `fraud_sql_analysis_report.docx` → SQL insights
* `fraud_dashboard.pbix` → Power BI dashboard
* `fraud_dashboard_report.docx` → Dashboard explanation
* `fraud_dataset_report.docx` → Dataset & data quality
* `Data_Pipeline_Report.docx` → Pipeline architecture
* `Final_End_to_End_Report.docx` → Complete project report

---

## 🔍 Key Insights

* Fraud mainly occurs in TRANSFER and CASH_OUT
* High-risk scores strongly correlate with fraud
* Balance inconsistencies are a major fraud indicator
* Dataset is highly imbalanced → important for modeling
